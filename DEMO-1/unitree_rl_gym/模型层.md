# actor_critic.py
ActorCritic
假设：

- 观测维度 `mlp_input_dim_a = 11`
- 隐藏层 `actor_hidden_dims = [256, 256, 128]`
- 动作维度 `num_actions = 3`
- 激活函数 `activation = nn.ReLU()`

## Actor（策略）网络
最终搭出来的网络结构

1. Linear(11 → 256)
2. ReLU
3. Linear(256 → 256)
4. ReLU
5. Linear(256 → 128)
6. ReLU
7. Linear(128 → 3)
## 构建Critic（价值）网络
与Actor的区别 ：输出维度为1 ：Critic输出单个标量价值V(s)
## 动作噪声参数
- 创建形状为 (num_actions,) 的参数，如 torch.ones(12) → [1.0, 1.0, ..., 1.0]
- nn.Parameter ：告诉PyTorch这是需要优化的参数
- 可学习的探索噪声 ：每个动作维度有自己的标准差，训练中会调整
为什么需要可学习的噪声？
- 初始时噪声大，充分探索
- 训练后期噪声减小，策略趋于确定性
- 比固定噪声更灵活

创建记忆模块
```python
self.memory_a = Memory(num_actor_obs, type=rnn_type, num_layers=rnn_num_layers, hidden_size=rnn_hidden_size)
```
`Memory(...)`

这是一个**记忆类**，内部一般是：
- **RNN**
- **LSTM**
- **GRU**
```python
input_a = self.memory_a(observations, masks, hidden_states)
return super().act(input_a.squeeze(0))
```
### ① **observations**

当前环境观测比如：位置、速度、角度

### ② **masks**

掩码（用来标记 episode 是否结束）= 相当于之前的 `dones`作用：**遇到 done，就把隐藏状态清零**

### ③ **hidden_states**

**上一步的隐藏状态（记忆）**= 智能体上一刻记住的东西
squeeze(0) ：去除序列维度，形状变为 (batch_size, rnn_hidden_size)，squeeze(0) 专门删除 第 0 维（最前面）大小为 1 的维度。
在强化学习 `act()` 决策时：**智能体每一步只看当前 1 帧**，所以我们输入的序列长度是：**seq_len = 1**于是输入形状变成：`
` (1, batch, feature)` 
## Memory 类RNN的核心实现，封装了LSTM/GRU的操作
####  1. 训练模式（Train Mode）

**就是智能体在 “学习、练级”**

### 特点

- 多环境并行（比如 8 个、16 个环境一起跑）
- 动作带**随机探索**（从正态分布采样）
- 需要**masks** 来重置 LSTM 记忆
- 需要**hidden_states** 保存每步记忆
- 会记录经验、计算损失、更新网络
- 会传 `masks` → `masks is not None` → `batch_mode=True`
#### 2. 推理模式（Inference / Eval Mode）

**就是智能体已经学好了，开始 “实战、表演”**

### 特点

- 通常**单环境**运行
- 动作**几乎不加噪声**（直接用分布均值 action_mean）
- 不探索、不学习
- 不需要 masks（结束直接用 reset ()）
- 不需要保存批量记忆
- **不传 masks** → `masks is None` → `batch_mode=False`
