# actor_critic.py
ActorCritic
假设：

- 观测维度 `mlp_input_dim_a = 11`
- 隐藏层 `actor_hidden_dims = [256, 256, 128]`
- 动作维度 `num_actions = 3`
- 激活函数 `activation = nn.ReLU()`

## Actor（策略）网络
最终搭出来的网络结构

1. Linear(11 → 256)
2. ReLU
3. Linear(256 → 256)
4. ReLU
5. Linear(256 → 128)
6. ReLU
7. Linear(128 → 3)
## 构建Critic（价值）网络
与Actor的区别 ：输出维度为1 ：Critic输出单个标量价值V(s)
## 动作噪声参数
- 创建形状为 (num_actions,) 的参数，如 torch.ones(12) → [1.0, 1.0, ..., 1.0]
- nn.Parameter ：告诉PyTorch这是需要优化的参数
- 可学习的探索噪声 ：每个动作维度有自己的标准差，训练中会调整
为什么需要可学习的噪声？
- 初始时噪声大，充分探索
- 训练后期噪声减小，策略趋于确定性
- 比固定噪声更灵活
## 权重初始化方法
````python
@staticmethod
def init_weights(sequential, scales):

"""

使用正交初始化初始化线性层的权重。

参数:

sequential: 包含线性层的nn.Sequential

scales: 每个线性层的增益值列表

"""

[torch.nn.init.orthogonal_(module.weight, gain=scales[idx]) for idx, module in

enumerate(mod for mod in sequential if isinstance(mod, nn.Linear))]
```
